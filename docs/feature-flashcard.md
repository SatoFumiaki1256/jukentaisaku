# フラッシュカード機能 設計メモ

> ステータス: 🔲 未実装（外骨格のみ index.html に存在）
> 担当ファイル（予定）: `flashcard.html`

---

## 概要

製図試験で頻出の「知識・用語・計算式」を短時間で反復学習できる  
フラッシュカードアプリ。スマホでスキマ時間に使えることを重視する。

---

## ユーザーストーリー

- 「敷地面積の計算方法がいつも怪しい」→ カードで繰り返し確認
- 「階段の最小幅は？」→ 瞬時に答えられるようになる
- 「不正解カードだけ絞り込んで再挑戦したい」

---

## カードデータ構造（案）

```json
{
  "id": "fc_001",
  "category": "法規",
  "subcategory": "採光・換気",
  "front": "居室の採光に必要な開口部の最小面積は？",
  "back": "床面積の **1/7以上**（住宅の場合）\n\n建築基準法 第28条",
  "tags": ["採光", "居室", "法規"],
  "difficulty": 2
}
```

### カテゴリ案
| カテゴリ | 内容例 |
|---|---|
| 法規 | 採光・換気・斜線・建蔽率・容積率 |
| 構造 | 木造の継手・仕口・金物 |
| 計算 | 面積・斜線・高さ制限の計算式 |
| エスキス手法 | ゾーニング・動線の定石 |
| 用語 | 建築用語の定義 |

---

## UI・操作フロー（案）

1. カテゴリ選択画面
2. カードをタップ → 答え表示
3. 「わかった ✅ / もう一度 🔁」で仕分け
4. セッション終了 → 正解率・苦手カード表示
5. 苦手カードだけ再挑戦モード

### カードデザイン
- 表面: 問題文（大きく、読みやすく）
- 裏面: 答え + 解説 + 条文番号（あれば）
- スワイプ操作：左 = もう一度、右 = わかった
- バウハウスカラー：表面 = ベージュ、裏面 = ホワイト

---

## データ管理

- カードデータ: `cards.json`（静的ファイルで管理）
- 学習進捗: localStorage（`fc_progress`）
  ```json
  {
    "fc_001": { "correct": 5, "incorrect": 2, "lastSeen": "2026-04-20" },
    ...
  }
  ```
